# Clay Model Upload Instructions

## Prerequisites

1. **HuggingFace Account**: Ensure you have access to the MVRL organization on HuggingFace
2. **HuggingFace Token**: Create a write token at https://huggingface.co/settings/tokens

## Option 1: Automated Upload (Recommended)

### Step 1: Set HuggingFace Token

Add your HuggingFace token as a secret in Cursor Dashboard:
- Go to: Cloud Agents > Secrets
- Add secret: `HF_TOKEN` = `your_token_here`

### Step 2: Run Upload Script

```bash
cd /workspace
python3 upload_to_hub.py
```

This will:
- Load the Clay model checkpoint
- Create the repository `MVRL/clay-v1.5` if it doesn't exist
- Upload the checkpoint file (`clay-v1.5.ckpt`)
- Upload the README with usage instructions

## Option 2: Manual Upload via CLI

### Step 1: Login to HuggingFace

```bash
huggingface-cli login
```

### Step 2: Create Repository

```bash
huggingface-cli repo create MVRL/clay-v1.5 --type model
```

### Step 3: Upload Files

```bash
cd /workspace
huggingface-cli upload MVRL/clay-v1.5 clay-v1.5-upload/clay-v1.5.ckpt
```

### Step 4: Upload README

The README is generated by the upload script. You can also create it manually on the HuggingFace website.

## Verification

After upload, verify the model is accessible:

```python
from rshf.clay import Clay

model = Clay.from_pretrained("MVRL/clay-v1.5")
print("Model loaded successfully!")
```

## Files to Upload

- `clay-v1.5.ckpt` (4.9GB) - The PyTorch Lightning checkpoint
- `README.md` - Model card with usage instructions

https://huggingface.co/MVRL/clay-v1.5
